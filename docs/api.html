API de Benefícios

Modelo de dados e operações CRUD — bip-teste-integrado

Base URL

http://localhost:8080/api/v1

Modelo de Dados — Tabela BENEFICIO

Coluna Tipo SQL Restrições Descrição
ID BIGINT PK · GENERATED ALWAYS AS IDENTITY Identificador único, gerado automaticamente pelo banco
NOME VARCHAR(100) NOT NULL Nome do benefício
DESCRICAO VARCHAR(255) — Descrição detalhada do benefício
VALOR DECIMAL(15,2) NOT NULL Valor monetário do benefício
ATIVO BOOLEAN DEFAULT TRUE Indica se o benefício está ativo
VERSION BIGINT DEFAULT 0 Controle de versão para optimistic locking (JPA @Version)

Representação JSON

{
  "id":       1,
  "nome":     "Beneficio A",
  "descricao": "Descrição A",
  "valor":    1000.00,
  "ativo":    true,
  "version":  0
}

Operações CRUD — /beneficios

GET /beneficios

Descrição: Retorna a lista de todos os benefícios cadastrados.

Parâmetros: nenhum

Resposta de sucesso

[
  {
    "id": 1,
    "nome": "Beneficio A",
    "descricao": "Descrição A",
    "valor": 1000.00,
    "ativo": true,
    "version": 0
  },
  {
    "id": 2,
    "nome": "Beneficio B",
    "descricao": "Descrição B",
    "valor": 500.00,
    "ativo": true,
    "version": 0
  }
]
200Lista retornada com sucesso (array vazio se não houver registros)
GET /beneficios/{id}

Descrição: Retorna um benefício pelo seu ID.

Path param: id obrigatório — identificador do benefício

Resposta de sucesso

{
  "id": 1,
  "nome": "Beneficio A",
  "descricao": "Descrição A",
  "valor": 1000.00,
  "ativo": true,
  "version": 0
}
200Benefício encontrado
404Benefício não encontrado para o ID informado
POST /beneficios

Descrição: Cria um novo benefício.

Content-Type: application/json

Request body

{
  "nome":      "Beneficio C",          // obrigatório
  "descricao": "Descrição do novo",    // opcional
  "valor":     750.00,                 // obrigatório
  "ativo":     true                    // opcional — padrão true
}

Resposta de sucesso

{
  "id": 3,
  "nome": "Beneficio C",
  "descricao": "Descrição do novo",
  "valor": 750.00,
  "ativo": true,
  "version": 0
}
201Benefício criado; Location header aponta para o novo recurso
400Dados inválidos (campo obrigatório ausente, valor negativo, etc.)
PUT /beneficios/{id}

Descrição: Atualiza todos os campos de um benefício existente.

Path param: id obrigatório

Content-Type: application/json

Request body

{
  "nome":      "Beneficio A Atualizado",
  "descricao": "Nova descrição",
  "valor":     1200.00,
  "ativo":     true,
  "version":   0    // obrigatório — optimistic locking
}

Resposta de sucesso

{
  "id": 1,
  "nome": "Beneficio A Atualizado",
  "descricao": "Nova descrição",
  "valor": 1200.00,
  "ativo": true,
  "version": 1
}
200Benefício atualizado com sucesso
400Dados inválidos
404Benefício não encontrado
409Conflito de versão (optimistic locking) — recurso foi alterado por outro processo
DELETE /beneficios/{id}

Descrição: Remove um benefício pelo seu ID.

Path param: id obrigatório

Parâmetros: nenhum no body

204Benefício removido com sucesso (sem conteúdo na resposta)
404Benefício não encontrado

Operação Extra — Transferência de Valor

POST /beneficios/transferencia

Descrição: Transfere um valor do saldo de um benefício para outro de forma atômica e segura (via EJB com optimistic locking e validação de saldo).

Content-Type: application/json

Request body

{
  "fromId": 1,       // obrigatório — ID do benefício de origem
  "toId":   2,       // obrigatório — ID do benefício de destino
  "valor":  200.00   // obrigatório — valor a transferir (> 0)
}
200Transferência realizada com sucesso
400Saldo insuficiente, valor inválido ou IDs iguais
404Um dos benefícios não encontrado
409Conflito de versão — tente novamente
500Erro interno; transação revertida (rollback automático pelo EJB)
